Case Study

Rewinder Accuracy Root-Cause Investigation

Investigated a production rewinder accuracy problem by validating control logic, machine behavior, and physical assumptions until the true source of systematic length error was isolated.

This was one of my first larger automation-related investigations in production. Although the final corrective action was implemented by the machine supplier, my role was to identify and structure the technical evidence needed to explain the problem clearly.

System Overview

The rewinder system measures tape length during production and depends on stable mechanical contact, correct PLC-side logic, and reliable assumptions about how rotation translates into linear movement. The investigation required separating software assumptions from physical machine behavior.

Simplified block diagram of the rewinder machine and tape path used to explain the investigation safely without exposing proprietary details.

Challenge

The factory was seeing systematic rewinder length inaccuracies. The challenge was to determine whether the error came from PLC logic, encoder interpretation, measurement assumptions, or physical machine behavior.

At this stage, it was important not to jump to conclusions. The main task was to verify each plausible source of error one by one and build a technically defensible explanation.

Engineering Approach

  • Control-side review: checked the rewinder logic and distance-calculation assumptions to determine whether the PLC program itself could explain the mismatch.
  • Independent validation: compared the rewinder’s reported length behavior against independent observation methods rather than trusting only machine output.
  • Mechanical reasoning: evaluated how top-wheel contact and changing effective diameter could influence real linear distance measurement.
  • Problem framing: structured the findings clearly enough that the machine supplier could review the issue and take responsibility for corrective action.

Key Finding

The investigation showed that the main issue was not a simple PLC bug. The root cause was tied to the machine’s physical measurement behavior — specifically the changing effective diameter / contact behavior of the top wheel during rewinding, which created systematic length error.

That finding shifted the problem from “software malfunction” to “measurement method and machine mechanics.”

Result

I prepared and presented the investigation summary to the machine supplier, Sanki. The final corrective action was then implemented on the supplier side through calibration-factor-based correction using a 1000 m reference tape.

My direct contribution was identifying the actual problem pattern, reducing ambiguity, and producing the technical explanation that allowed supplier-side corrective work to move forward.

What This Project Demonstrates

  • Ability to investigate production problems without assuming the PLC is always the cause
  • Mechanical reasoning applied to automation and measurement systems
  • Structured communication of technical findings to external machine suppliers
  • Early evidence of systems-level troubleshooting in a manufacturing environment
